Database Management System

Data refers to raw facts and figures that are unprocessed and have no inherent meaning. Information, on the other hand, is processed data that is meaningful and useful. It is data that has been organized or structured in a way that is useful for decision-making.

Features of Information

The key features of information include accuracy, relevance, completeness, and timeliness. It should be presented in a manner that is understandable and usable for the recipient.

Database and its Purpose

A database is a systematic collection of data that can be easily accessed, managed, and updated. The purpose of a database is to store and organize data efficiently, allowing users to retrieve and manipulate data as needed.

Terminologies in Database

  • Table: A collection of data organized in rows and columns.
  • Field: A single piece of data or attribute within a table (i.e., a column).
  • Record: A complete set of related fields, typically represented by a row in a table.
  • Tuple: A single row in a relational database table, representing a record.
  • Object: A collection of related data and methods that operate on that data, often used in object-oriented databases.
  • Keys: Attributes or sets of attributes that uniquely identify a record in a table.

Data Dictionary

A data dictionary is a centralized repository that contains detailed information about data elements within a database. It describes the structure, relationships, and constraints on the data.

Database Management System (DBMS)

Introduction

A Database Management System (DBMS) is software that provides an interface to interact with the database. It manages data, allows for queries, and ensures data integrity.

Objectives

The main objectives of a DBMS are data integrity, security, consistency, and efficiency in managing large volumes of data.

Advantages

Advantages of DBMS include reduced data redundancy, improved data security, easier data retrieval, and improved data consistency.

Disadvantages

Disadvantages include cost of implementation, complexity, and the need for specialized knowledge.

Types of Database Model

  • Hierarchical Model: Data is organized in a tree-like structure where each record has a single parent.
  • Network Model: Data is organized in a graph structure, allowing multiple relationships between records.
  • Relational Model: Data is stored in tables with relationships between them.
  • Entity-Relationship Model: Uses entities and relationships to represent data.

Integrity Constraints and Types

  • Domain: Specifies the valid values for an attribute.
  • Entity: Ensures that each entity (record) is unique.
  • Referential: Ensures that foreign keys in a table refer to valid primary keys in another table.

Normalization

Introduction

Normalization is the process of organizing data in a database to reduce redundancy and improve data integrity.

Normal Forms

  • 1NF: Ensures that each column contains atomic (indivisible) values.
  • 2NF: Ensures that the database is in 1NF and that all non-key attributes are fully dependent on the primary key.
  • 3NF: Ensures that the database is in 2NF and that there are no transitive dependencies between attributes.

Advantages

Normalization helps in reducing redundancy and ensures data consistency. It also optimizes database performance.

Disadvantages

Normalization can lead to more complex queries and sometimes performance issues due to the need for multiple joins.

Centralized and Distributed Database

Introduction

A centralized database stores all data at a single location, while a distributed database stores data across multiple sites or servers.

Advantages

Centralized databases offer easier management, while distributed databases provide improved fault tolerance and availability.

Disadvantages

Centralized databases may be a single point of failure, whereas distributed databases can be more complex to maintain and manage.

Comparison

Centralized databases are simpler to manage but more prone to failure, while distributed databases provide better performance but are more complex to maintain.

Database Security

Introduction

Database security involves protecting the database from unauthorized access, attacks, and other security threats.

Challenges

Challenges include data breaches, insider threats, and maintaining data confidentiality and integrity.

Security Measures

Measures include encryption, access control, regular audits, and implementing strong authentication methods.

Roles of DBA

The Database Administrator (DBA) is responsible for managing and maintaining the database, ensuring data integrity, and enforcing security policies.

Practical Topics

DDL and DML Languages

Data Definition Language (DDL) is used for defining the database schema, while Data Manipulation Language (DML) is used for querying and modifying data.

SQL Data Types

  • CHAR: A fixed-length character string.
  • VARCHAR: A variable-length character string.
  • BINARY: A fixed-length binary data type.
  • VARBINARY: A variable-length binary data type.
  • TINYBLOB: A small binary data type.
  • TINYTEXT: A small text data type.
  • TEXT: A text data type.
  • LONGTEXT: A long text data type.
  • ENUM: A list of predefined values.
  • BIT: A binary data type used to store boolean values.
  • TINYINT: A small integer data type.
  • BOOLEAN: A boolean data type, often represented as 1 (true) or 0 (false).
  • INTEGER: A standard integer data type.
  • FLOAT: A floating-point number data type used for precise calculations.
  • DOUBLE: A double-precision floating-point number data type.
  • DECIMAL: A fixed-point number used for exact numeric values, often used for monetary values.
  • DATE: A date value, formatted as YYYY-MM-DD.
  • DATETIME: A date and time value, formatted as YYYY-MM-DD HH:MM:SS.
