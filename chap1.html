Chapter 1: Database Management System

Data refers to raw facts and figures, while information is processed, organized, and meaningful for decision-making. A database is a systematic collection of data, designed for efficient storage, retrieval, and update. A Database Management System (DBMS) is software that enables users to define, create, maintain, and control access to the database.

Features of Information

Quality information must be accurate, relevant, complete, and timely. These features ensure that data can be relied upon for effective decision-making.

Database and its Purpose

Databases store large volumes of data in structured forms, enabling efficient querying, reporting, integrity, security, and easy backup.

Terminologies in Database

  • Table: A set of records (rows) and fields (columns).
  • Field: A single attribute or column in a record.
  • Record: A complete row of related data.
  • Tuple: Same as a record in relational databases.
  • Object: A data structure used in object-oriented or object-relational systems.
  • Key: Attribute(s) that uniquely identify records.

Data Dictionary

A centralized repository that stores metadata—definitions, structures, constraints, relationships, and user privileges for all data elements in the database.

DBMS: Objectives, Advantages & Disadvantages

Objectives

  • Ensure data integrity, consistency, and accuracy
  • Provide data security and controlled access
  • Support multi-user environments and concurrency
  • Enable efficient data retrieval and management

Advantages

  • Reduced redundancy and improved data quality through normalization
  • Enhanced data security with user authentication and privileges
  • Improved data sharing across applications and users
  • Centralized backup and recovery mechanisms
  • Powerful query capability for complex information retrieval

Disadvantages

  • High cost of implementation, licensing, and maintenance
  • Increased hardware and training requirements
  • System complexity that requires specialized personnel
  • Potential performance overhead for small datasets

Database Models

  • Hierarchical Model: Organizes data in a tree structure. Efficient for one-to-many relationships but rigid and difficult to manage many-to-many relationships.
  • Network Model: Uses graph-like structures. Supports many-to-many relationships but complex to design and navigate.
  • Relational Model: Stores data in tables using rows and columns. Uses keys to define relationships, offers flexibility and ease of use, but can be slower for complex joins.
  • Entity-Relationship Model: Conceptual design using entities and relationships. Useful for modeling but not directly implemented in systems.

Integrity Constraints

  • Domain: Defines the permissible values for an attribute.
  • Entity Integrity: Ensures a primary key has no NULL values.
  • Referential Integrity: Ensures that foreign keys in one table correspond to primary keys in another, preserving relationship consistency.

Normalization

Normalization is the process of organizing tables to minimize redundancy and improve data integrity. It involves decomposing tables into smaller, related tables.

  • 1NF: Ensures all columns contain only atomic values and each row is unique.
  • 2NF: Achieved when 1NF is satisfied and there are no partial dependencies on a composite primary key.
  • 3NF: Achieved when 2NF is satisfied and there are no transitive dependencies among non-key attributes.

Normalization improves consistency and storage efficiency, but excessively normalized schemas may require complex joins or impact performance.

Centralized vs Distributed Databases

  • Centralized: Stores data in one physical location. Easier to manage, secure, and back up, but vulnerable to single points of failure.
  • Distributed: Data is distributed across multiple servers/sites. Offers better performance, fault tolerance, and availability but is more complex and expensive to manage.

Database Security

Security measures include user authentication, authorization, encryption, auditing, and maintaining strong access controls to protect data confidentiality and integrity.

Roles of Database Administrator (DBA)

  • Database design, implementation, and schema updates
  • Granting user privileges and enforcing security policies
  • Performing backups, recovery, and disaster planning
  • Monitoring performance and optimizing queries and storage

SQL Languages

  • DDL: Defines or alters database schema (CREATE, ALTER, DROP)
  • DML: Manipulates data (INSERT, UPDATE, DELETE, SELECT)
  • DCL: Manages permissions (GRANT, REVOKE)
  • TCL: Controls transactions (COMMIT, ROLLBACK)

Data Types in SQL

  • CHAR, VARCHAR
  • BINARY, VARBINARY
  • TINYBLOB, TINYTEXT, TEXT, LONGTEXT
  • ENUM, BIT, BOOLEAN
  • TINYINT, INTEGER, FLOAT, DOUBLE, DECIMAL
  • DATE, DATETIME

Q9: Explain primary key, foreign key, candidate key.

  • Primary Key: Unique identifier attribute for a table.
  • Foreign Key: Attribute that links to a primary key in another table.
  • Candidate Key: Any attribute or combination that can uniquely identify records.

Q10: What is SQL? Components?

Structured Query Language used for schema definition (DDL), data manipulation (DML), permissions (DCL), and transaction control (TCL).

Q12: Definitions of key terms

  • Data Dictionary: Repository of metadata.
  • Primary Key: Unique row identifier.
  • Relationship: Association between tables.
  • DML: INSERT, UPDATE, DELETE.
  • SQL: Query language.
  • Data Integrity: Correctness and consistency.
  • DDL: CREATE, ALTER, DROP.
  • Data Security: Protecting data assets.
  • Database System: Combination of database, DBMS software, and users.
